## 2024-03-07 - Math.random() for secure paths
**Vulnerability:** Weak random generation using Math.random() for storage file paths.
**Learning:** Found multiple instances where Math.random() was used for file paths in file uploads (e.g. `const fileName = \`\${Math.random()}.\${fileExt}\`;`). This results in predictable file names. Although they might be protected by RLS, using `crypto.randomUUID()` ensures a cryptographically secure, collision-free, and unpredictable string format which is much safer for potentially sensitive uploaded assets. The memory section notes this explicitly: "The project uses crypto.randomUUID() for generating secure, unique file paths for storage uploads (e.g., in Settings.tsx, Onboarding.tsx, GroupSettings.tsx, and PollDialog.tsx) to avoid the security risks associated with predictable names generated by Math.random()."
**Prevention:** Always use `crypto.randomUUID()` instead of `Math.random()` when generating identifiers or storage paths for security-sensitive objects.
