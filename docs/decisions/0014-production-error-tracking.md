# 0014 — Production error tracking via Sentry (client-side, errors-only)

**Status:** accepted (2026-07-22) — fleet-wide policy, template ADR-0008

**Context.** plano ships to production unattended: PRs auto-merge on green (ADR-0005) and only a nightly heavy tier watches the pipeline (ADR-0006). Nothing watched *production itself* — a runtime error a real user hit left no trace the owner could see. The `@sentry/react` SDK was already wired in the code (`src/lib/sentry.ts` `initSentry`, called from `src/entry.client.tsx`; `setSentryUser` in `src/root.tsx`; `AppErrorBoundary` / `WidgetErrorBoundary` / `MapErrorBoundary` route boundary reporting) but it is an env-gated no-op unless `VITE_SENTRY_DSN` is set — and the production DSN was **unset until 2026-07-22**, so capture was effectively off (principle 8, "the system watches production, not just the pipeline", was failing in practice).

**Decision.** Sentry is plano's production error tracker. The DSN is provided at build time via **`VITE_SENTRY_DSN`** (a Vercel Production environment variable; set 2026-07-22 and verified live — a test exception reaches the project's ingest endpoint). `initSentry` stays a **no-op in dev and whenever the DSN is unset**, so local and preview runs never emit. Configuration is deliberately **frugal (principle 6): errors only** — `tracesSampleRate: 0`, both replay sample rates `0` — no performance tracing, no session replay. `beforeSend` scrubs `token` / `access_token` query params from breadcrumb URLs before send. Capture is **client-side only**; server/SSR (loaders, actions, resource routes) capture is a deliberate later option, not part of this decision. `setSentryUser` attaches the authenticated user id (no PII beyond the id) so errors are attributable.

**Rejected alternative.** A full APM tier (tracing + session replay) — rejected as disproportionate cost for a solo-owner project (principle 6); sampling can be raised later if a concrete need appears. Self-hosting Sentry or a bespoke error sink — rejected as operational burden that contradicts "boring, handed-over" (principle 11) when the hosted free tier covers the need.
